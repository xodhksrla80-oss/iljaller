🔍 윈도우 업데이트 오류, 왜 자꾸 생길까?

윈도우 업데이트가 자꾸 실패하는 데는 의외로 다양한 이유가 있더라고요. 인터넷 연결이 불안하거나, 디스크 공간이 부족하거나, 시스템 파일이 손상된 경우도 많고요. 보안 프로그램과 충돌하는 경우도 꽤 흔해요. 그래서 오류 코드를 잘 확인해 보면, 원인 파악이 훨씬 쉬워져요!

업데이트 오류 주요 원인
5가지
캐시 손상 / 디스크 공간 부족 / 시스템 파일 오류 / 방화벽 차단 / 서비스 미실행
⚠️
업데이트, 왜 미루면 위험할까?

윈도우 업데이트에는 중요한 보안 패치가 포함돼 있어서, 깜빡 잊고 오래 미루면 해킹이나 바이러스 감염 가능성이 확 올라가요. 혹시라도 오류가 뜬다면, 너무 오래 두지 말고 바로 해결하는 게 안전하겠죠?

🔧 가장 먼저 해볼 것 — 업데이트 문제 해결사

복잡하게 머리 아프게 굴리기 전에, 윈도우 안에 내장된 문제 해결사 먼저 한 번 돌려보세요. 생각보다 이걸로 바로 고쳐지는 경우, 진짜 많아요!

1
설정 앱 열기: Win + I

키보드 Windows 키와 I키를 동시에 누르면 설정 앱이 바로 열립니다.

2
시스템 → 문제 해결 → 기타 문제 해결사

목록에서 "Windows Update"를 찾아 "실행" 버튼을 클릭하세요.

3
자동 진단 완료 후 안내에 따라 조치

윈도우가 알아서 문제를 찾아주고, 뭐가 잘못됐는지 해결해 줘요. 다 끝나면 다시 업데이트 시도~ 이 방법으로 금방 해결되는 경우 의외로 많더라고요.

4
재시작 후 업데이트 재시도

문제 해결사 돌리고 나면, 꼭 컴퓨터 한 번 재시작하고 업데이트 다시 눌러보세요! 깜빡하고 바로 업데이트했다가 또 오류 뜨는 경우가 많아요😂

❌ 자주 나오는 오류 코드별 해결법

그래도 계속 안 된다면, 화면에 뜨는 오류 코드 한 번 확인해보세요! 아래에서 대표적인 것들 정리해봤어요. 거의 대부분 여기서 해결할 수 있습니다.

0x80070005
접근 거부 오류 — 권한 문제
해결법: 관리자 권한으로 다시 실행해 보세요. 그리고 혹시 백신 프로그램 쓰고 있다면, 잠깐 꺼두고 시도해보는 것도 좋아요. 타사 백신이 업데이트 폴더 접근 막을 때가 많거든요.
0x800705b4
업데이트 서비스 연결 시간 초과
해결법: 일단 인터넷 연결이 정상인지 확인! 라우터를 재시작하는 것도 방법이에요. Windows Update 서비스를 재시작해 주세요(아래에 방법 나와있어요). 만약 서버가 잠깐 바쁜 거라면, 몇 시간 후 다시 시도하면 되는 경우도 많아요.
0x80073712
파일이 없거나 손상됨
해결법: 명령 프롬프트(관리자 권한)에서 'sfc /scannow', 'DISM /Online /Cleanup-Image /RestoreHealth' 차례로 입력! 시스템 파일이 고장난 경우 이 두 가지 명령어로 대부분 복구가 가능합니다.
0x8007000D
데이터가 유효하지 않음 — 캐시 손상
해결법: Windows Update 캐시를 싹 초기화해 주세요(아래에 자세히 설명 있어요). 임시 파일에 문제가 생겼을 때 이렇게 해주면 새로 내려받으면서 정상적으로 작동하곤 해요.
0x80240034
업데이트 다운로드 실패
해결법: 인터넷 연결 다시 확인하고, DNS를 8.8.8.8 같은 걸로 변경해 보는 것도 도움돼요. 혹시 VPN이나 프록시를 쓰고 있다면 그거 잠깐 꺼두고 시도해보세요. 네트워크 문제일 때가 제일 많거든요.
ℹ️
오류 코드 확인 방법

설정 → Windows Update → 업데이트 기록에서 실패한 항목을 클릭하면, 오류 코드가 친절하게 뜹니다. 위에서 소개한 목록에서 해당 번호 찾아 해결해 보시면 돼요!

🗑 Windows Update 캐시 초기화

업데이트 오류의 절반 이상이, 사실은 임시로 저장된 파일(캐시) 때문에 생기는 경우가 많아요. 이럴 때 캐시 폴더만 깔끔하게 지워주면, 윈도우가 새롭게 파일을 다시 받아오면서 오류도 없어지더라고요. 순서대로 해볼게요!

1
명령 프롬프트를 관리자로 열기

시작 버튼 우클릭 → "터미널(관리자)" 또는 검색창에 "cmd" 입력 후 "관리자로 실행" 선택.

2
업데이트 서비스 중지 명령어 입력

net stop wuauserv 입력 후 Enter → net stop bits 입력 후 Enter. 업데이트 관련 서비스를 일시 중지합니다.

3
캐시 폴더 삭제

파일 탐색기에서 C:\Windows\SoftwareDistribution\Download 경로로 들어가서 거기 있는 파일들 싹 다 지워줍니다. 이 폴더는 업데이트 임시 파일만 모아두는 곳이에요. 내 중요한 파일에는 전혀 영향 없어요!

4
서비스 재시작 및 업데이트 재시도

명령 프롬프트로 돌아가서 net start wuauserv → net start bits 입력. 이후 설정 → Windows Update에서 다시 업데이트 확인.

✅
절대 무서워하지 마세요!

이 캐시 폴더는 시스템 필수 파일을 건드리는 게 아니라, 진짜 그때그때 임시로만 쓰는 공간이에요. 지워도 완전히 안전합니다. 다음에 업데이트하면 다시 데이터가 생겨요, 걱정 마세요!

💾 디스크 공간 부족 & 수동 업데이트 방법

윈도우 업데이트할 땐 생각보다 많은 여유 공간이 필요해요. C 드라이브가 꽉 찼다면 업데이트가 아예 안 되거나 진행 중에 멈출 수 있어요. 최소한 20GB 정도는 꼭 비워두시길 추천드려요!

업데이트 종류필요 공간비고
보안 패치 (월별)500MB~2GB매달 정기 업데이트
기능 업데이트8~20GB연 2회 대규모 업데이트
드라이버 업데이트100~500MB선택적 업데이트

📥 자동 업데이트 안 될 때 수동으로 하는 방법

1
Microsoft Update 카탈로그 접속

브라우저에서 catalog.update.microsoft.com 으로 접속. 업데이트 번호(예: KB5034441)로 검색해서 직접 다운로드 가능합니다.

2
Windows 11 설치 미디어 도구 활용

마이크로소프트 공식 사이트에서 "미디어 만들기 도구"를 다운로드해서 실행. "이 PC 지금 업그레이드"를 선택하면 인터넷으로 직접 최신 버전으로 업데이트됩니다.

🎯
업데이트 후에도 여전히 느리다?

그럴 땐 "윈도우11 느려졌을 때 해결법" 관련 글도 한 번 참고해보시면 좋아요. 특히 시작 프로그램 정리하고, 전원 옵션 바꿔주면 체감 속도가 빠릿빠릿 빨라집니다!

🎯 핵심 요약
  • 먼저 내장 문제 해결사: 설정 → 시스템 → 문제 해결 → Windows Update 실행
  • 오류 코드 0x80070005: 백신 일시 중지 + 관리자 권한 확인
  • 오류 코드 0x80073712: sfc /scannow + DISM 복구 명령어 실행
  • 반복 실패: SoftwareDistribution\Download 폴더 삭제 후 재시도
  • C 드라이브 여유 공간 최소 20GB 이상 확보 필수
  • 자동 안 될 때: Microsoft Update 카탈로그에서 수동 다운로드
